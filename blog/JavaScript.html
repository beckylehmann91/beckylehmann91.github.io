Ruby vs. JavaScript

A brief overview of the syntactic differences between Ruby and JavaScript

by Becky Lehmann

November 16, 2015

Overview

If you have some exposure with Ruby and JavaScript, you've probably noticed several syntactic differences between the languages. Similar, I imagine, to trying to learn Spanish and French at the same time, the crossover between the Ruby and JavaScript can be pretty confusing. While you could take a much deeper dive into each of the topics I will cover (and you certainly should!), the purpose of this post is to give a high-level overview of some of those key syntactic differences.

Variable Declaration

In Ruby, variable declaration is pretty straighforward, following a format of variable_name = assigned_value. In JavaScript, the declaration looks a bit different: 

      
        var variable_name = assigned_value
As you can see, you must precede the variable name with "var" in order to indicate it is a variable. JavaScript uses this same format, regardless of the scope of the variable. You determine the scope not through the syntax of the variable name, but by looking at where the variable is defined (if variable_name is declared within a function, it is local to that function; if it is declared outside of any functions, it is considered global). The same concept of scope exists in Ruby. However, local variables look different from instance and global variables. For example, if I wanted to declare the same variable_name within a method, but make it accessible to other methods within the same class, I would declare it as @variable_name. Thus, JavaScript draws scope purely from where the variable is declared, while Ruby depends more on the format of the variable name itself.

Ruby Hashes and JavaScript Objects

In Ruby, you may see a hash constructed like so: 

      
        my_hash = {
          key_one: value_one,
          key_two: value_two,
          key_three: value_three
      }
The syntax looks pretty much the same in JavaScript, except that you would use the variable naming conventions described above. One of the main syntactic differences between hashes and objects relates to how values within the hash are accessed. In Ruby, you would use my_hash[:key_one] to retrieve the value, value_one. In JavaScript, the more common notation is with a dot, like so: my_hash.key_one.

Methods

The main syntactic difference between methods in Ruby and JavaScript relates to the use of parenthesis when calling a method. Let's say I have a string "Becky" assigned to the variable my_name. In Ruby, if I wanted to convert the string to all uppercase letters and assign it to the variable upcase_name, I would do the following: 

      
        my_name = "Becky"
        upcase_name = my_name.upcase
In JavaScript, you must use parenthesis when calling a function. Thus, a similar method in JavaScript would look like this: 
      
        var my_name = "Becky"
        var upcase_name = my_name.toUpperCase();
As you can see, I've included a set of parenthesis after "toUpperCase," even though there is no argument included. In JavaScript, you must include parenthesis for many of these built-in methods, otherwise the code will not run properly.

Summary

While this is just a start to the syntactic differences between Ruby and JavaScript, it is important to note the details so you don't run into issues while writing your code. For all you Rubyists out there, beware of the JavaScript semicolons! They may trip you up along the way.
